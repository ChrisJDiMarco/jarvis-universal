# Skill: Weekly Review

## Trigger
"weekly review", "how's the week looking", "end of week", "week in review", "Friday wrap-up", "wrap up the week"

## Goal
Synthesize the week's performance, measure progress against current goals, update memory, and set sharp priorities for the next 5 days.

## Phase 1: Pull Data (run concurrently)

Gather from all relevant sources based on operator's archetype:

**Universal:**
- Calendar: meetings held, calls that were wins/losses, no-shows
- Memory check: read `logs/daily-activity.md` for the week
- Inbox: count items processed vs. still pending

**Business / Freelancer:**
- New leads or inquiries this week
- Proposals or quotes sent
- Deals closed or projects completed
- Active clients/projects and their status
- Revenue logged this week vs. target

**Creator:**
- Content pieces published vs. planned
- Top-performing piece (engagement, views)
- Content backlog size

**Developer / Builder:**
- Features shipped
- PRs merged
- Bugs resolved vs. new ones opened

**Student / Researcher:**
- Assignments completed
- Chapters/sections written or read
- Progress on primary project

## Phase 2: Progress Scoreboard

```
=== WEEK [N] OF [PERIOD] — [Date Range] ===

📊 KEY METRIC: [primary goal metric]
  This week:   [X]
  Target pace: [X per week]
  Cumulative:  [X] / [total target]
  Status:      [ON TRACK / BEHIND / AHEAD]

📋 COMPLETED THIS WEEK:
  [Bulleted list of actual completions]

❌ PLANNED BUT NOT DONE:
  [Honest list of slipped items]
```

## Phase 3: What Worked / What Didn't

Three questions:
1. **What moved the needle this week?** (specific action → specific result)
2. **What did I do that I should stop doing?** (time sinks, low-ROI activities)
3. **What should I do more of next week?** (high-ROI patterns to double down on)

Source answers from: `logs/daily-activity.md`, memory files, calendar review.

## Phase 4: Top 5 Priorities for Next Week

```
NEXT WEEK — TOP 5
1. [Specific outcome] by [day] — [why it matters]
2. ...
3. ...
4. ...
5. ...
```

Rules for priority setting:
- Each priority must have a clear "done" signal
- At least 1 priority must be a follow-up on last week's top priority
- No more than 2 priorities per major context
- Urgent tasks rank above important-but-not-urgent

## Phase 5: Memory Updates

1. Check if any learnings emerged worth persisting → update `memory/learnings.md`
2. Update `memory/context.md` with current project status
3. Update `memory/decisions.md` if major decisions were made
4. Log the weekly review to `logs/daily-activity.md`

## Phase 6: Output

Save to `owners-inbox/weekly-review-[YYYY-MM-DD].md`

Present to operator:
1. Progress scoreboard (visual, fast to scan)
2. Top insight from the week (1–2 sentences)
3. Top 5 next-week priorities (numbered, crisp)
4. One question: "Anything from the week you want to talk through before we close it out?"

## Rules
- Be honest about off-target weeks — no spin
- Keep entire output to one page / under 500 words
- Always end forward-looking, not retrospective
- If data is incomplete: estimate and flag gaps, don't skip the review
