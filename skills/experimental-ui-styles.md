# Skill: Experimental UI Styles

## Trigger

"design this", "make it look interesting", "make it creative", "pick a visual
direction", "experimental UI", "aesthetic direction", "less generic", "make this
feel designed", any website/app/dashboard/prototype/slide task where visual
style is open.

## Goal

Use the operator's experimental UI style catalog as a creative direction layer before
building or redesigning visual interfaces.

## Canonical Skill

Codex-discoverable skill:
`.agents/skills/experimental-ui-styles/SKILL.md`

Full style catalog:
`.agents/skills/experimental-ui-styles/STYLE_CATALOG.md`

Load the canonical skill first, then read the catalog when visual direction is
open or the interface risks looking generic.

## Process

1. Frame the surface: product, audience, workflow density, tone, constraints.
2. Browse the catalog and pick 3-5 candidate styles.
3. Choose one primary style and, optionally, one secondary accent.
4. Translate the style into typography, layout, color, surface, texture, and
   interaction decisions.
5. Build the usable experience; do not let novelty break accessibility,
   responsiveness, scanability, or project conventions.
6. Name the final style recipe in the handoff when it is useful.

## Rules

- Treat the catalog as a mood board, not a costume rack.
- Use sensitive aesthetics as metaphors only unless the product domain matches.
- Keep the product/workflow clear in the first viewport.
- Prefer one strong direction over a pile of unrelated effects.
