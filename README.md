## Why This Exists

Every AI tool puts you in the orchestrator seat вАФ you decide which model, what context to paste, what to carry across sessions, when to switch tools. For a system that's supposed to be working _for_ you, that's a lot of operator work.

JARVIS flips it. You say what you want in plain language. An orchestrator routes the request to the right specialist. The specialists have their own tools, their own memory, and their own rules. The system extracts a lesson from every failure and never makes the same mistake twice.

It's not an assistant. It's a team that happens to live inside Claude Code.

## рЯІ† The Memory System

JARVIS never asks the same question twice. Memory is stored in structured, capped markdown files вАФ loaded lazily so they don't bloat every conversation.

```
Session Start
вФВ
вФЬвФАвФА L0 вАФ Identity (~2,000 tokens) ¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј ALWAYS loaded
вФВ   вФФвФАвФА Who you are, archetype, working style
вФВ
вФЬвФАвФА L1 вАФ Critical Facts (~1,500 tokens)  ALWAYS loaded
вФВ   вФФвФАвФА Active focus, constraints, preferences
вФВ
вФЬвФАвФА L2 вАФ Domain Context ¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј Loaded when relevant
вФВ   вФЬвФАвФА memory/context.md   вЖТ projects, clients, tools
вФВ   вФЬвФАвФА memory/decisions.md вЖТ past decisions + rationale
вФВ   вФФвФАвФА memory/learnings.md вЖТ extracted patterns
вФВ
вФФвФАвФА L3 вАФ Deep History ¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј¬Ј Loaded on recovery only
    вФФвФАвФА relationships, domain files, soul.md
```

**At session end**, JARVIS evaluates what it learned and writes updates to the appropriate layer. The index rebuilds automatically.

Caps are calibrated for Opus 4.7 (L1 5k chars, context 25k, learnings 20k, ai-intelligence 25k). The cap is a forcing function for dropping dead entries вАФ not a context-window guard.

```bash
# Semantic search via Ollama embeddings (cosine similarity), with transparent BM25 fallback
python3 memory/memory_search.py "what did we decide about pricing" --top 3
python3 memory/semantic_search.py "anything about onboarding flow" --index-path skills/learned/learned_index.json --top 5
```

## рЯЪА Setup

### Prerequisites
- [Claude Code](https://claude.com/product/claude-code) вАФ requires Claude Pro or API key

### Install

```bash
git clone https://github.com/ChrisJDiMarco/jarvis-universal.git ~/jarvis
cd ~/jarvis
bash setup/install.sh        # checks deps, sets permissions, validates structure
claude                       # launch JARVIS вАФ first run triggers the wizard
```

The installer is idempotent вАФ re-run anytime. To verify everything is healthy later: `bash setup/check.sh`. To run the full test suite alongside the health check: `bash setup/check.sh --full-suite`.

If JARVIS doesn't trigger the first-run wizard automatically for any reason, type `"run first-run wizard"` in your first session вАФ that forces the setup conversation.

JARVIS detects first run automatically and walks you through 3 questions:

```
JARVIS: What's your name, and in one sentence вАФ what do you do?
JARVIS: What's your main goal right now?
JARVIS: What tools do you use day-to-day?
```

After about two minutes, JARVIS has populated memory, activated the right agents, and is ready to work.

### Try These First

Six prompts that exercise the full system. Type one and watch the orchestrator route it.

```
"morning briefing"
"research [a topic you care about] вАФ deep dive with sources"
"write a short LinkedIn post about [recent thing you learned]"
"build a landing page for [imaginary product] вАФ make it animated"
"review the last commit in this repo"
"what would you change to ship this faster?"
```

Once a few sessions are in, run `bash scripts/dashboard.sh` and open `owners-inbox/dashboard.html` for a status snapshot.

### Versioning, updating, recovery

JARVIS Universal follows [SemVer](https://semver.org/) starting with v1.0.0. To pin to a specific version: `git checkout v1.0.0`. To pull a new release safely: see [`docs/UPDATING.md`](docs/UPDATING.md). When something breaks: see [`docs/RECOVERY.md`](docs/RECOVERY.md). Release process: [`docs/RELEASING.md`](docs/RELEASING.md). Full release history: [`CHANGELOG.md`](CHANGELOG.md).

### Optional Power-Up: Semantic Code Search

Add a fully local vector index over your entire JARVIS corpus вАФ code, memory, skills, agents, docs. JARVIS can then find things by meaning, not exact keywords. Great for queries like `"find where we handle [concept]"` or `"anything about [topic]?"` across every project.

Fully offline: Ollama embeddings + local Milvus container. Zero API cost, data stays on your machine.

**One-shot install:**
```bash
bash setup/install-semantic-search.sh
```

The installer is idempotent: checks for Docker вЖТ installs Ollama if missing вЖТ pulls the embedding model вЖТ spins up the Milvus container вЖТ registers the MCP вЖТ runs the first index. ~10 minutes total, ~1.5 GB disk. Use `--check` to see the current state without installing anything, or `--yes` to skip confirmations.

Once installed, the Stop hook auto-triggers incremental reindex when source files change вАФ no manual maintenance. Full architecture details: [`docs/semantic-code-search-setup.md`](docs/semantic-code-search-setup.md).

<summary><b>рЯФђ Deep Research</b></summary>

Ask for a deep dive and you get a 6-phase pipeline, not a one-shot answer:

1. **Scope** вАФ JARVIS restates the question and flags ambiguities
2. **Search** вАФ 6-10 parallel queries across sources
3. **Screen** вАФ filter to high-signal sources, discard junk
4. **Extract** вАФ pull claims, stats, quotes with citations
5. **Synthesize** вАФ build the narrative, flag gaps
6. **Deliver** вАФ 1,500вАУ3,000 word report with source list + confidence levels

Claims that can't be verified get flagged with `[unverified]` instead of being silently presented as fact.

## рЯТ° Philosophy

> *Five principles that drive every design decision.*

**1 вАФ Route, don't execute.** The orchestrator never does domain work directly. It delegates to specialists. Context stays clean, outputs stay high-quality.

**2 вАФ Memory is sacred.** The system is only as good as what it remembers. Memory files are capped, structured, and maintained carefully вАФ so information stays current and never bloats.

**3 вАФ Self-improve or stagnate.** JARVIS extracts lessons from every failure. The MetaClaw system turns mistakes into permanent rules. It's not just an assistant вАФ it's a learning system.

**4 вАФ Plan before executing.** For any task over 3 steps, JARVIS proposes a plan first. For significant work, it writes a Requirement Document. Wasted execution is worse than wasted planning.

**5 вАФ The most direct tool wins.** Direct MCP over browser automation. Specific API over general search. Fastest reliable path вАФ always.
